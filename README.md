# Завдання №17 
Умова: Американці вважають нещасливим число 13, китайці - 4. Для спільних навчань штаб української армії вирішив не використовувати бойову техніку, номер якої містить числа 4 або 13 (40123, 13313, 12345 або 13040), щоб не бентежити іноземних колег. В розпорядженні армії 100000 одиниць бойової техніки. Визначити скільки всього номерів доведеться виключити.
## Код файлу Exercise

```java
package domain;

public class Exercise {
    public static int Calculate(int [] milit_numbs){
        int counter = 0;        
        for(int i = 0; i < 100000; i++){
            String numb = Integer.toString(i);
            if(numb.contains("4") || numb.contains("13")){
                counter++;
            }
        }
        return counter;
    }
}
```
## Код файлу TestResult

package test;

## Трохи власногоо пояснення

Отже. ми маємо два класи, тестовий, та головний, де ми виконуємо метод вирішення задачі. Але спершу тест клас; тут я створюю масив із 100000 підрозділами, та виводжу у консоль повідомлення із викликом методу. У метод calculation передаю атрибут milit_numbs щоб ніи порівнювати номер підрозділу із умовою. Чесно можна було виконати задачу і без масивів, просто як порівнюючи числа без створення окремого масиву, але додав масив, бо за темою потрібно. За самою умовою, відбираються тільки ті числа, які мають у собі 4 та 13, бо вони не везучі, але я не врахував у коді випадки де може бути 31 , типу може хтось іще буде вважати це число нещасливим, але я хз.
